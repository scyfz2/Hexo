ISV入驻与发布审核

Fan2025-09-182025-09-18

国际化电商ISV入驻和应用程序发布的审核

1. 账号注册与基础资料

站点 用途 关键入口
TikTok for Developers
https://developers.tiktok.com		 创建开发者身份、获取 Client Key、阅读全局 API 条款 右上角「Get Started」→ 填写邮箱+PIN → 设置密码即可开通开发者账号

2. 电商类 ISV 专属后台（重点）

站点 用途 关键入口
TikTok Shop Partner Center
https://partner.tiktokshop.com		 • 提交企业资质 & 电商应用
• 申请 Sandbox 店铺做交易测试
• 查看审核进度、补件、申诉
• 上线后查看调用量、GMV、商家评分		 登录后顶部导航「App Development」→「Create App」即进入电商应用工单系统

注：虽然域名带「partner」，但技术型 ISV 与代运营型 TSP 共用同一套账户体系；系统会自动根据你选择的「Partner Type = Developer」展示技术表单而非服务表单。

3. 审核工单与内部对接（实习生日常）

字节内部使用 「Gondola」 作为跨境 B 端审核引擎，前端页面仍然挂在
https://partner.tiktokshop.com/admin/audit
只是普通开发者看不到「Admin」路径。实习生有额外权限，登录后会出现：

  • Application Queue（应用审核池）
  • Risk Flag Console（合规风控标记）
  • IGT Treasury（ISV Growth Token 发放与回收）

所有操作记录都会回写到同一工单号，因此对外 ISV 看到的「进度条」与内部 Gondola 状态实时一致，可避免重复沟通。

4. 上线后的运维 & 数据

站点 用途 关键入口
同上 Partner Center • 查看 API 调用监控
• 订阅 webhook（订单、退款、店铺失效）
• 发布新版并触发灰度审核		 「My Apps」→ 选中应用 →「Analytics / Webhooks / Versions」

5. 辅助资源（文档&社群）

  • 文档根目录：https://developers.tiktok.com/doc/overview
  • 电商专用 API 手册：在 Partner Center 右上角「Docs」→「TikTok Shop API Reference」
  • 开发者 Slack 邀请链接：登录 Partner Center 后自动弹出，频道 #isv-onboarding-cn 有中文值班同学。

一张图看懂「什么时候该进哪个站」

结论

  • 对外只有一个入口记忆点：partner.tiktokshop.com
  • 审核后台叫 Gondola，挂在同一域名下，实习生应该权限可见
  • 非电商类（内容、广告、登录）才需要跑去 developers.tiktok.com
